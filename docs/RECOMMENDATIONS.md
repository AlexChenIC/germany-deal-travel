# Personalized Recommendations

`为我推荐` 页面使用确定性规则，不依赖外部 AI 调用。它基于当前静态数据、家庭画像和浏览器里的排除列表实时生成。

## 家庭画像

- 常驻柏林。
- 3 位成年人 + 1 位约 11 个月宝宝，可兼顾长辈同行。
- 优先级：避暑、低折腾、宝宝友好、推车/室内/水边/泳池、柏林市内、柏林出发、全包和接送。

## 推荐分组

- 近期柏林活动：柏林市内活动，优先新鲜、家庭友好、低交通成本。
- 周边避暑与短住：柏林周边/德国境内，优先波罗的海、温泉/SPA、泳池、水边、家庭酒店和短住。
- 远期阳光/全包：土耳其、埃及、希腊、西班牙等方向，优先全包、接送、直飞和 5-9 天时长。
- 邮轮与大型旅行观察：欧洲/近德国邮轮优先，短航线和餐食省心线索加分。
- 儿童活动建议：儿童咖啡馆、开放活动、家庭中心、低龄音乐项目、游泳课和室内活动。

## 排序和解释

每条旅行推荐由以下因素综合排序：

- 原始 `priorityScore`。
- `familyScore`。
- 分组内的额外加分，例如全包、接送、泳池、SPA、海边、短住、低价、新鲜度。
- 风险降权，例如成人限定、CSD/party/club/nightlife 等可能拥挤或偏成人的活动、长线旅行、超高价。

每张卡会显示：

- 匹配分。
- 推荐原因。
- 注意事项，例如空调/泳池需核对、婴儿登船年龄、航班行李和婴儿票、人流/热天风险。
- 收藏和排除按钮。

## 当前限制

- 第一版不做实时可订性和实时价格检查。
- 酒店空调、泳池、评价风险还没有逐条核验；这些会在 `避暑短住` 页面进一步处理。
- 推荐逻辑是规则式的，适合稳定可解释；后续可以在规则结果上叠加摘要或更细的用户偏好。
