# Implementation Steps

这是“柏林家庭出行雷达”的执行清单。原则是：每一步尽量小，完成后构建、提交、部署，再进入下一步；除非用户明确调整优先级。

## Step 0: 基线整理

状态：已完成。

目的：确保仓库从干净、可构建的状态继续扩展。

任务：
- 检查 worktree 是否有中断开发留下的半成品。
- 清理 `src/App.tsx` 中未完成的推荐页草稿。
- 运行 `npm run build`。
- 在开始新功能前确认 `git status --short` 没有无关代码改动。

完成标准：
- 构建通过。
- 没有半成品 UI 留在主应用。
- 仓库有清晰的基线提交。

## Step 1: 路线图和执行计划

状态：已完成。

目的：把已经讨论过的优化方向固化为可执行任务，避免后续一次混入太多功能。

任务：
- 扩展 `docs/ROADMAP.md`，整理产品和工程路线图。
- 新增 `docs/IMPLEMENTATION_STEPS.md`，记录具体步骤、任务和完成标准。
- 在 README 中加入开发路线入口，方便后续回看。

完成标准：
- 路线图和执行步骤已经提交。
- 不引入运行时行为变化。

## Step 2: 每日自动检索可视化

状态：已完成。

目的：让已经存在的每日自动更新更明显、更可诊断。

当前状态：
- `.github/workflows/pages.yml` 已经支持 push、手动触发和每日 cron。
- workflow 已经在部署前运行 `npm run update-data`。

任务：
- 在页面上增加自动化状态模块。
- 展示 `radar.generatedAt`、来源成功/失败/跳过数量，以及缓存回退提示。
- 在条目上增加“今日新增”和“本周新增”标签。
- 文档化柏林本地时间下的定时任务触发时间，以及如何手动运行 workflow。
- 决定 cron 是否需要调整到更适合柏林早晨查看的时间。

完成标准：
- 用户不打开 GitHub Actions 也能知道最近一次数据更新是否健康。
- 来源失败和缓存回退可见。
- 构建通过并部署。

## Step 3: 为我推荐页面

状态：已完成。

目的：根据 Alex 家庭情况，自动挑出值得关注的近期活动和远期旅行优惠。

家庭画像：
- 常驻柏林。
- 3 位成年人和 1 位约 11 个月宝宝。
- 重点偏好：避暑、宝宝友好、长辈友好、低折腾、推车方便、短住时优先泳池/SPA、长途优先 BER 出发。

任务：
- 新增 `为我推荐` tab。
- 创建确定性推荐分组：
  - 近期柏林活动。
  - 周边避暑和短住。
  - 远期阳光/全包优惠。
  - 邮轮和大型旅行观察。
  - 儿童活动建议。
- 每条推荐显示推荐原因。
- 复用收藏和排除动作。
- 将推荐逻辑放到可测试的独立模块中。

完成标准：
- 页面可以基于当前静态数据运行。
- 每条推荐都解释为什么出现。
- 收藏/排除会影响推荐结果。
- 构建和浏览器冒烟测试通过。

## Step 4: 避暑短住页面

状态：已完成。

目的：解决高温周末在柏林 1-2 小时范围内找冷气可靠、适合宝宝和长辈的酒店问题。

任务：
- 新增 `避暑短住` 页面。
- 定义酒店候选结构：
  - 酒店名、位置、交通时间、预订链接。
  - 房间空调证据。
  - 泳池、桑拿、SPA、湖/河边资源。
  - 宝宝适配和风险。
  - 价格区间和检查时间。
  - 评价风险，尤其是房间闷热或空调差。
- 先录入第一批人工核验候选。
- 增加空调、泳池、SPA、预算和距离筛选。

完成标准：
- 页面足以支持一次真实周末避暑决策。
- 证据和不确定性都明确展示。
- 没有实时查价时，不宣称实时可订。

当前第一版：
- 已新增 `避暑短住` tab。
- 已录入 8 个柏林市内/近郊/2 小时内候选。
- 已支持可靠空调、泳池、SPA、宝宝优先、车程范围和排序筛选。
- 每个候选展示空调证据、泳池/SPA证据、宝宝适配、优点、风险和回源链接。

增强状态：
- 已新增每日天气触发推荐，使用 Open-Meteo 未来 16 天预报。
- 已新增价格/可订性状态接口，支持可选 `SERPAPI_API_KEY`。
- 已新增近期评论风险接口，支持可选 `GOOGLE_PLACES_API_KEY`。
- 没有 API key 时，页面明确显示未配置并提供检查入口。

## Step 8: 地图增强

状态：8.1-8.3 已完成。

目的：在控制成本和 API key 风险的前提下提高地图实用性。

任务：
- 保留所有地点的 Google Maps 外部跳转链接。
- 保留 Maps Embed API 作为可选单地点内嵌地图。
- 评估 Leaflet/OpenStreetMap 作为免费多点筛选地图。
- 只有在确实需要 Google 原生多点地图体验时，再使用 Google Maps JavaScript API。

完成标准：
- 地图能支持筛选后的多点浏览。
- 没有 API key 时仍有可用回退。

当前完成：
- Step 8.1：儿童活动页已接入 Leaflet + OpenStreetMap 多点地图，支持缩放和平移，不依赖 Google Maps JavaScript API。
- Step 8.2：地图点位、地图下方地点 chip 和活动卡片实现双向联动；点击点位会高亮并滚到卡片，点击卡片“上方地图”会聚焦地图。
- Step 8.3：新增区县筛选、参考点选择和按参考距离排序；距离为直线估算，仅用于快速判断大致区域。
